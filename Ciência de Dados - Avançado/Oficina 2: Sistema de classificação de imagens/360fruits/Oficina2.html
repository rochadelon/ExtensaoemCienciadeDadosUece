1. Introduo

Este projeto visa desenvolver um sistema de classificao automtica de imagens de frutas (mas, bananas e laranjas) utilizando tcnicas de aprendizado profundo. O objetivo principal criar um modelo capaz de identificar com alta preciso diferentes tipos de frutas, mesmo em condies variadas de iluminao, orientao e escala.

Atravs da implementao de transfer learning com a arquitetura MobileNetV2 e tcnicas avanadas de aumento de dados, o projeto busca superar os desafios comuns em classificao de imagens, como conjuntos de dados limitados e desbalanceados. Esta abordagem possibilita o desenvolvimento de um classificador robusto e eficiente que pode ser aplicado em diversos contextos, desde sistemas automatizados de verificao de qualidade at aplicativos mveis para identificao de frutasO objetivo do projeto criar um sistema capaz de receber uma imagem de uma fruta e retornar a sua classificao como sendo uma maa, banana ou laranja, para isso necessrio criar um modelo de aprendizado supervisionado capaz de reconhecer caracterticas das frutas pretendidas por meio de um banco de imagens pr-rotuladas.

Este modelo utilizar tcnicas de aumento de dados significativas para melhorar a generalizao e o desempenho da classificao, mesmo com um conjunto de dados potencialmente limitado.

Cada parmetro do ImageDataGenerator tem um papel especfico na criao de variaes das imagens para treinamento do modelo:

  • rescale=1./255: Normaliza os valores dos pixels, convertendo-os da faixa 0-255 para 0-1, facilitando o processamento pela rede neural.

  • rotation_range=40: Aplica rotaes aleatrias nas imagens de at 40 graus (positivos ou negativos), ajudando o modelo a reconhecer objetos em diferentes orientaes.

  • width_shift_range=0.3: Desloca horizontalmente as imagens em at 30% de sua largura, tornando o modelo mais robusto quanto posio horizontal dos objetos.

  • height_shift_range=0.3: Desloca verticalmente as imagens em at 30% de sua altura, tornando o modelo mais robusto quanto posio vertical dos objetos.

  • shear_range=0.3: Aplica transformaes de cisalhamento (distoro angular) nas imagens, ajudando o modelo a lidar com diferentes perspectivas.

  • zoom_range=0.3: Aplica zoom aleatrio de aproximao ou afastamento em at 30%, permitindo que o modelo reconhea objetos em diferentes escalas.

  • horizontal_flip=True: Inverte aleatoriamente as imagens horizontalmente, til para reconhecimento de objetos que podem aparecer em ambas orientaes.

  • vertical_flip=True: Inverte aleatoriamente as imagens verticalmente, til para reconhecimento de objetos que podem aparecer de cabea para baixo.

  • brightness_range=[0.7, 1.3]: Varia aleatoriamente o brilho das imagens entre 70% e 130% do original, tornando o modelo mais robusto a diferentes condies de iluminao.

  • validation_split=0.2: Reserva 20% das imagens para validao, permitindo avaliar o desempenho do modelo durante o treinamento.

Este cdigo configura trs geradores de dados para treinar e avaliar o modelo de classificao de imagens:

  • train_generator: Cria lotes de 32 imagens da pasta de treinamento, aplicando todas as transformaes de aumento de dados definidas anteriormente. Usa 80% dos dados do diretrio de treino (devido ao subset='training').

  • val_generator: Gera dados de validao durante o treinamento, usando os 20% restantes das imagens do mesmo diretrio de treino. Tambm aplica as mesmas transformaes de aumento de dados.

  • test_generator: Carrega imagens do diretrio de teste para avaliao final do modelo, apenas redimensionando e normalizando-as (sem aumento de dados). A opo 'shuffle=False' mantm a ordem original das imagens para facilitar a anlise dos resultados.

Todos os geradores utilizam os parmetros pr-definindos:

  • Redimensionam as imagens para 100100 pixels
  • Usam modo 'sparse' para codificar os rtulos como nmeros inteiros (0, 1, 2)
  • Carregam as imagens em lotes de 32 para processamento eficiente

class_weights um dicionrio que atribui diferentes pesos s classes durante o treinamento do modelo.

  • Classe 0: Apple Recebe peso 1.0 (parece ser a classe majoritria com 4569 amostras)

  • Classe 1: Banana Recebe peso aproximadamente 3.2 (45691430), compensando sua menor representao

  • Classe 2: Orange Recebe peso aproximadamente 3.87 (45691181), compensando sua menor representao

Este balanceamento importante porque:

  • Impede que o modelo favorea a classe majoritria
  • Fora o modelo a prestar mais ateno s classes minoritrias
  • Melhora o desempenho geral em conjuntos de dados desbalanceados

O valor de cada peso calculado como a razo entre o nmero de amostras na classe majoritria (4569) e o nmero de amostras na classe respectiva, garantindo que todas as classes tenham influncia proporcional durante o treinamento.

class_weights = {0: 1.0, 1: 4569/1430, 2: 4569/1181}

Este momento crucial pois configura o modelo base MobileNetV2 para transfer learning:

  • base_model: Utiliza a arquitetura MobileNetV2 pr-treinada no dataset ImageNet, que eficiente e otimizada para dispositivos mveis.

  • input_shape=(100, 100, 3): Define o tamanho das imagens de entrada como 100x100 pixels com 3 canais de cores (RGB), mantendo consistncia com o tamanho de imagem definido anteriormente.

  • include_top=False: Remove as camadas de classificao originais do MobileNetV2, permitindo adicionar camadas personalizadas para a classificao especfica de frutas.

  • weights='imagenet': Carrega os pesos pr-treinados no ImageNet, aproveitando caractersticas j aprendidas como deteco de bordas, texturas e padres.
